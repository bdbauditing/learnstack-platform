# Part 5 — Test Management + Final Capstone

## The scenario

TaskForge Inc. is about to ship version 2.0 of its task-management web app. You are the QA lead. The release date is set. The dev team says it is ready. Now it is your job to prove it — or stop it.

These 8 exercises are a real release gate sequence. You start before a single test is written (requirements review) and finish with a signed sign-off document that either sends the build to production or pulls the brakes. Every exercise feeds into the next, so do them in order.

## Prerequisites

- Completed Parts 1–4 (or solid experience with manual testing, API testing, and Playwright)
- Node.js 20+
- A text editor
- Familiarity with YAML and CSV formats

## How to use this part

1. Read `concepts.md` first — plain-English explanations of every idea you will use.
2. Take the `quiz.yaml` quiz to check your understanding.
3. Work through exercises in order — each one is a checkpoint in the release process.
4. Each exercise has a `README.md` (start here), a `starter/` folder, and a `spec.md` that defines exactly what the grader checks.
