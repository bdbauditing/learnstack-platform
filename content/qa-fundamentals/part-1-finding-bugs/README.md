# Part 1 — Finding and Reporting Bugs

**Time estimate:** ~3 hours (reading + exercises)
**Prerequisites:** None — this is the starting point.

## What you'll learn

By the end of this part you can:
- Tell the difference between a bug report that gets fixed and one that gets ignored
- Write reproduction steps a developer can follow blindly
- Classify bugs by severity and priority without confusing the two
