# Part 2 — Test Design Techniques

**Time estimate:** ~4 hours (reading + exercises)
**Prerequisites:** Part 1 — Finding and Reporting Bugs

## What you'll learn

By the end of this part you can:
- Write test cases that actually catch bugs (not just describe happy paths)
- Apply equivalence partitioning and boundary value analysis systematically
- Build a decision table and a state transition matrix
- Select the right tests for regression, smoke, and exploratory sessions
- Link tests to requirements via a traceability matrix
